## Phase Goal

**As a** desenvolvedor que consome a lib, **I want to** declarar campos de schema com açúcares ergonômicos (`@BsonType`, `@Description`, `@Pattern`, `@Optional`, `@Enum`, `@Min`/`@Max`, `@MinLength`/`@MaxLength`) e aninhar classes decoradas/arrays, **so that** eu escrevo schemas expressivos sem cair no `@Prop({...})` genérico a cada campo, compilando para o mesmo `ModelValidationSchema` do objeto plano.

> Fatia vertical (Wave 2, paralela a 06-02): adiciona a ergonomia de declaração de campo e o aninhamento — capacidades reais de DX sobre a base de 06-01.

<objective>
Entregar o conjunto completo de açúcares composáveis sobre `@Prop` (D-02) e o compile recursivo de schemas aninhados/arrays (D-05), mantendo a equivalência byte-a-byte com o objeto plano (DECO-03). Ao fim, um dev declara um schema rico e aninhado só com decorators.

Purpose: `@Prop` genérico sozinho (06-01) é verboso; os açúcares e o aninhamento são o que torna a API de decorators genuinamente ergonômica e completa (DECO-01 pleno, DECO-03 recursivo).
Output: `src/schema/sugars.ts` com os 8+ açúcares, compile recursivo em `src/schema/compile.ts`, exports no barrel, e testes de composição/aninhamento.
</objective>

## Trust Boundaries

|----------|-------------|
| declaração de schema (açúcares/aninhamento) → validator compilado | Um açúcar ou compile recursivo incorreto gera um `$jsonSchema` mais fraco que o pretendido, enfraquecendo a validação server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-03-01 | Tampering / sub-validação | campo sem decorator fica fora do schema compilado (D-04) — dev pode achar que um campo está validado quando não está | medium | mitigate | comportamento coberto por teste + a ser documentado MUITO explicitamente nos guias (Fase de docs); teste de exemplo demonstra o comportamento |
| T-06-03-02 | Tampering | mapeamento de açúcar para chave JSON Schema errada (ex.: `min` em vez de `minimum`) produziria restrição ignorada pelo servidor | medium | mitigate | teste unit por açúcar verificando a chave exata + equivalência byte-a-byte ao objeto plano |
| T-06-03-03 | Tampering | compile recursivo que não propaga `additionalProperties: false` para subschemas permitiria props extras aninhadas | low | accept | `schemaValidatorBuilder`/`includeAdditionalPropertiesFalse` já recursiona `additionalProperties: false` sobre o schema aninhado no Model — compile só entrega o shape; coberto pelo teste de paridade Mongo-real (06-02) |
