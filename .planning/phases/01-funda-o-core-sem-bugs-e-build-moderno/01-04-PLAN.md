<objective>
Corrigir os bugs da camada Database dentro da arquitetura Proxy/registry existente (constraint do autor): (1) o `KModelProxyHandler` deve vincular métodos ao `target` (instância crua), não descartar o `.bind()` (QUAL-01 — bug de binding); (2) remover o fallback silencioso de dbName de teste (`mongoat-test`) e lançar `MongoatError` quando nenhum nome de banco for configurado (D-08); (3) adicionar uma API mínima de reset do registry `@internal` (D-09). Opcionalmente corrigir o duplo-Proxy do `defineModel()` deprecated (Open Question 3). Cada correção travada por teste de regressão.

Purpose: A camada Database é a raiz do dispatch de métodos (Proxy) e da conexão. O fix de binding afeta TODO método do Model; o fix de dbName elimina um risco de escrita em banco não intencional; o reset do registry destrava testes isolados nesta e na próxima fase.
Output: `src/database/index.ts` corrigido + testes de proxy-binding, dbname-required e registry-reset verdes.
</objective>

<user_story>
**As a** desenvolvedor usando Mongoat, **I want to** que os métodos do meu model mantenham o contexto correto e que a lib falhe alto quando eu esquecer de configurar o nome do banco, **so that** eu não perco `this` silenciosamente nem escrevo acidentalmente num banco `mongoat-test` em produção.

> Slice: cada task é teste-de-regressão + fix + verde, para um comportamento observável distinto (binding, dbName, reset).
</user_story>
