<objective>
Corrigir os quatro bugs restantes de QUAL-01 na camada Model, todos em `src/model/index.ts`, dentro da arquitetura existente: (1) `insertMany` deve aguardar os pre-hooks com `Promise.all` antes de inserir; (2) `find()` deve ter tipo de retorno `Promise<WithId<ModelType> | null>` consistente (sem a união síncrona e o `?? null` morto); (3) `schemaValidatorBuilder` deve clonar o schema com `structuredClone` antes de mutá-lo, para não vazar entre models; (4) acesso à collection antes da conexão deve lançar `MongoatError` via um helper `getCollectionOrThrow()` (D-10). Além disso, tornar o registro de model atômico com detecção de config divergente (D-06), mantendo o constructor síncrono (D-07). Cada correção travada por teste; happy-path CRUD por método público (D-12).

Purpose: Fecha os bugs de correção da camada Model — o núcleo do valor da lib (CRUD confiável). Depende do plan 04 (registry reset + Proxy corrigido) para testes de registro isolados e para exercitar métodos via Proxy.
Output: `src/model/index.ts` corrigido + 6 arquivos de teste de regressão/CRUD verdes.
</objective>

<user_story>
**As a** desenvolvedor usando Mongoat, **I want to** que `insertMany` aplique meus pre-hooks, que `find()` tenha um tipo honesto, que schemas compartilhados não se contaminem e que operar antes de conectar falhe com um erro claro, **so that** o CRUD da lib seja confiável e previsível.

> Slice por bug: teste de regressão que falha → fix cirúrgico → verde. A última task adiciona o happy-path CRUD por método público.
</user_story>
