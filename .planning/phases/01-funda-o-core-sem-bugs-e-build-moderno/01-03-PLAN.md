<objective>
Montar a infraestrutura de teste da Fase 1 (D-12/D-13): `vitest` com resolução dos path aliases do projeto e um helper de `@testcontainers/mongodb` que sobe um MongoDB real em Docker, expõe a connection URI aos testes e o encerra no teardown. Purpose: Sem esta infra, os testes de regressão dos 5 bugs (plans 04 e 05) não têm onde rodar. Este plan é a scaffolding de verificação de QUAL-01 — os testes que provam cada fix dependem do container e da resolução de aliases estarem funcionando.
Output: `vitest.config.ts`, `test/setup/testcontainer.ts` e um smoke test verde contra Mongo real.
</objective>

<user_story>
**As a** mantenedor do Mongoat, **I want to** rodar testes de regressão contra um MongoDB real e descartável, **so that** cada bug corrigido nesta fase fica travado por um teste que falha se ele regredir.

> Slice: config do runner (aliases) → helper do container (Mongo real) → smoke verde. Após este plan, escrever um teste de regressão é trivial (`import { Model } from '@/model'` + usar a URI do container).
</user_story>
