Conectar a aplicação de plugins LOCAIS (`plugins[]`) no construtor do `Model`, satisfazendo PLUG-01 e PLUG-03. Inserir uma única chamada a `applyPlugins(this, Model[kGlobalPlugins], props.plugins ?? Output: Construtor do `Model` aplicando plugins locais antes do wrap do Proxy; 4 arquivos de teste (VALIDATION) verdes.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| autor-do-plugin → construtor do Model | O `setup()` roda síncrono durante `new Model(...)`; o construtor delega a `applyPlugins` (fail-loud) e nunca expõe referência viva de schema/validator/allowedMethods. |
| chamador → `new Model(...)` | Config de model (incl. `plugins[]`) é entrada confiável do desenvolvedor, não input de usuário final; a superfície de risco é o CÓDIGO do plugin, não dados. |

## STRIDE Threat Register (ASVS L1, block on high)

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-01 | Tampering | static de plugin colidindo com privado interno (`rawInsert`) no construtor | high | mitigate | `applyPlugins`/`registerPluginStatic` lança `STATIC_COLLISION` na construção contra o conjunto COMPLETO de nomes reservados; nunca sobrescreve lógica de driver-wrapping. Coberto por `test/model/plugins-static-collision.test.ts`. |
| T-07-02 | Elevation of Privilege | `setup()` de plugin de terceiro executa código arbitrário síncrono na construção (supply-chain in-process) | high | accept | Limitação fundamental do modelo de plugin in-process — não sandboxável (mesma classe de risco de qualquer dependência npm). Parte CONTROLÁVEL mitigada: `PluginContext` nunca expõe referência viva de schema/validator/allowedMethods (D-03), então mesmo um `setup()` malicioso não altera o `$jsonSchema` compilado. Disciplina de auditoria de supply-chain documentada; zero deps novas nesta fase. |
| T-07-04 | Denial of Service (config) | model meio-configurado registrado após erro de plugin | high | mitigate | Fail-loud (D-10): erro no `setup()` aborta `new Model(...)` antes de `registerModel`; nenhum model parcial no registry. Coberto por `test/model/plugins-fail-loud.test.ts` (`db.getModel` → `undefined`). |
| T-07-03 | Information Disclosure | `onHookError` de hook de plugin vaza `ctx` | low | accept | Herdado da Fase 2/3 sem mudança — `defaultOnHookError` loga só `err`, nunca o `ctx`; plugins reusam o mecanismo existente, sem nova superfície. |
